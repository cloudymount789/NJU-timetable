# 课表 App 需求文档

本文档与当前高保真原型（`nju-timetable.pen`）及迭代说明（`prompts.md`）对齐，描述**已实现于主原型**的能力、**与早期设想不一致的裁剪**，以及**暂缓/扩展稿**中的能力。

---

## 一、核心基础功能

### 1. 课程录入与编辑

- **手动添加课程**：录入课程名称、任课教师、上课教室、适用周次（单周/双周/隔周/指定周次等）、上课节次、星期几等；界面与 App 整体风格统一。
- **是否显示在课表**：支持将课程标记为不在周课表中展示（适用于网课、自学等），课程仍保留在资料与相关能力中。
- **教务导入**：支持对接南京大学教务系统，需要通过南京大学统一认证登录后读取教务网的个人课表并导入。
- **多源导入**：支持课表截图 OCR 导入；OCR 需指定截图对应的教学周范围，系统批量生成课程。
- **多份课表**：支持维护多份课表；列表通过「新建课表」统一管理，编辑后以胶囊等形式展示每份课表名称。
- **周课表上的入口**：主课表页右上角提供加号菜单，含 **导入课表**、**界面修改**（进入设置/外观相关能力）、**全部课程** 等（与原型一致）。
- **复杂排课规则**：单周、双周、隔周、指定周次等规则在录入侧可配置。
- **课程颜色**：支持为课程设置独立颜色标记（调出色轮自选）；默认为不同课程生成不同颜色；可在设置中重新随机生成配色方案
- **课程颜色配色方案和风格**：高级，极简，学术感，不刺眼，低饱和度莫兰迪色系，柔和马卡龙，轻微灰调，避免荧光色和高饱和色；冷暖色均匀，相邻课程颜色区分明显但不冲突，视觉感受清爽舒适，简约现代
- 
- **课程冲突显示**：课表内课程冲突时左右分栏。
- **全部课程**：独立列表展示所有课程（含不显示在周课表中的课程）；课程名下以灰色小字展示时间、地点、教师等信息；点击进入**课程详情**。
- 课表界面点击某个课程方框同样可进入课程详情界面

### 2. 课表视图

- **主视图**：周视图，展示一周内课程排布；与**标准栅格/隐形网格**对齐，课程块对齐节次与星期列；左侧支持扩展至 **12 节课**及对应作息（含第 7 节及以后时段规则，与 `prompts.md` 中节次时间约定一致）。
- **视觉风格**：课程块为**圆角毛玻璃/液态玻璃**质感，保证浅色与深色模式下文字可读；**不再**采用「整页背景图全覆盖课表」方案，课表页与全局其他页面风格一致（见 `prompts.md` 第 3 轮调整）。
- **周切换**：支持切换教学周、跳转任意周；课表标题区展示月份、第几周等信息，支持「我的课表」与**下拉选择查看谁的课表**（默认打开自己的，重进app后永远先显示自己的课表）。
- **顶栏**：状态栏区域预留；**加号、设置**等与标题同处顶栏区域；加号菜单与加号位置靠近并处于顶层；加号与课表下拉箭头具备**展开/收起**双状态示意。
- **课程块信息**：块内主标题为课程名，**第一行灰色小字为地点**（可按原型收窄列间距、加宽块宽）。
- **底部导航**：主导航调整为 **课表 / 功能 / 待办**；「我的」页已删除；待办页新增底部胶囊；其余二级页不展示底部胶囊。
- **返回**：各二级页左上角保留返回；返回控件位置**下移**，为系统状态栏预留空间。

### 3. 学期日历（功能 → 日历）

- **入口**：功能页中「日历」进入（原型中与「学期总览」对应的月历界面）。
- **月历**：展示整月日期格，**一天一格**；支持**左右箭头切换月份**；图例说明考试、放假、重要日等色块含义。
- **标注样式**：考试、放假、重要日等为**实色填充**；**今日**为**深灰描边**样式（非整块实色填充），「今日」信息条可为白底强调（与原型迭代一致）。
- **月标题行**：**月份与左右切换箭头**与中间日期说明同排布置（与早期「月份单独一行」方案不同，以原型为准）。
- **日历下方**：展示**当日日期**（居中加粗等样式）；提供「添加重要日」等操作（加号为胶囊按钮）。
- **当日待办**：在月历区域下方展示**当天带有时间信息的待办**（含持续时间区段、DDL 等示意），与待办模块联动。
- **当日标注展示与擦除**：选中带有**考试 / 放假 / 重要日**等标记的日期时，在下方信息区展示对应文案（与图例类型一致）。在该展示区域**下方**提供**「擦除特殊日期标记」**按钮：用于清除用户为当日配置的手动标记（重要日等）。若考试日期由「考试与作业」模块自动同步至日历，擦除行为须与数据模型一致（例如仅移除日历层覆盖、或提示需到考试与作业中删除源数据），实现时需在交互与文案上区分，避免用户误以为已删除考试记录。
- **系统同步的放假**：若放假数据来自只读假期数据，仍可擦除，但是在擦除前进行弹窗询问

### 4. 时间设置

- 支持自定义作息与每节课起止时间；上午/下午/晚间划分；与设置中的节次、开学日等配置一致（见第六节）。

## 二、实用增强功能

### 5. 上课提醒

- 课前 **5 / 10 / 15 / 30 分钟** 等提醒粒度。
- 提醒方式：通知、静音、震动、铃声等可选。
- **周末与法定节假日**不触发（可配置）。
- 支持**按课程设置**是否提醒。
- 列表项以**可编辑胶囊**等形式展示（与原型一致）。

### 6. 课程详情页
注：该界面一切信息均由用户自行填写。

- **成绩构成**：平时、期中、期末等占比。
- **评价维度**：如给分松紧、风格、 workload、考核方式、作业量等，分行展示。
- **资料**：快捷打开已上传文件；可自行上传。**编辑模式**下每条资料文件名右侧显示**主题色小叉号（×）**，点击后移除该文件（可二次确认，与实现策略一致）；浏览模式下不展示删除图标。
- **作业与考试**、**课程网站链接**（外链跳转）、**教师信息**（办公室、答疑时间、邮箱等）、**备注**。
- **编辑模式**：右上角 **铅笔**进入编辑，编辑中切换为 **打勾**表示完成编辑。
- **课程论坛**：先不做课程社区，因为需要弄数据库和服务器太麻烦了，主原型中**不单独提供**进入讨论区的入口；完整「课程社区 / 讨论详情 / 发布讨论」见**扩展稿**（第三节第 1 条）。
- **删除课程**：页面**最底部**提供**删除课程**入口；执行前**二次确认**，确认后从课表、全部课程及相关关联（待办/考试作业关联字段等按实现策略级联或提示）中移除或解除。

### 7. 考试 / 作业管理
具体考试/作业内容由用户自行编辑

- 考试信息、倒计时；作业截止与重复任务；相当于是与课程关联的待办整合。
- 支持按完成状态筛选、排序（与列表 UI 一致）。
- 考试与作业页右上角新增 **+ 新建** 主题色胶囊按钮。
- 新增「添加考试/作业」界面：可选择添加类型（考试/作业）；考试可编辑时间、地点、关联课程、考试类型（期中/期末/小测等自填）、备注；作业可设置 DDL（可按周几和具体时间，支持多天）、关联内容、是否重复（滑钮开关）。
- 新增「考试/作业详情」界面：点击列表项进入，右上角可编辑。
- **删除**：详情页**最底部**提供删除入口（如「删除本条记录」）；**二次确认**后删除并返回列表，若该项已同步至待办则按联动规则同步移除或标记。

### 8. 待办

- **列表页**：筛选 **全部 / 未完成 / 已完成**；**排序**为一行弱对比小字：**默认**（重要程度高且更紧急优先）与 **手动排序**，选中项使用**主题色**字体，未选中为次要色，无胶囊背景。
- **顶栏**：**批量编辑**（描边胶囊）+ **新建**（主题色胶囊，与日志等一致）。
- 点击批量编辑后，列表行右侧统一换成一个红底圆形垃圾桶ui的删除。若排序方式为手动排序，则可长按0.5s拖拽改变待办前后顺序
- **列表行右侧**：
  - **有持续时间**：展示**起始时间**（使用主题色），不强制展示完成按钮（与原型一致）。
  - **有 DDL**：展示 **DDL 文案** + **完成**按钮（**主题色填充**、白字）。
  - **普通待办**：仅 **完成**按钮（与 DDL 行同款主题色填充）。
- 列表行左侧使用数字标注待办顺序
- **已完成**：标题**划线**并弱化；在**全部**视图中，已完成项排在**列表最底部**。
- 待办列表新增可勾选项 **「关联考试与作业」**：开启后自动将考试/作业同步到待办；后续新建考试/作业条目也自动同步。

### 9. 待办详情

- 右上角 **主题色胶囊**：编辑/设置本条待办的**主题色**（带 palette 类图标）。
- 字段：**标题**、**内容**、**地点**。
- **待办种类（单选）**：三选一且选中打勾——① 有持续时间（展示起始/结束时间）；② 有 DDL（展示 DDL）；③ 普通（不展示时间）。
- **重要程度**：三档图形化选择（如 一般 / 重要 / 紧急）。
- **是否提醒**：使用与**设置页一致**的**滑动开关**（轨道与滑块样式统一变量）；开启后可填**提醒时间**。
- **进度日志**：时间线式记录；条目中时间展示**精确到秒**（与存储一致）。
- **删除待办**：详情页**最底部**提供**删除待办**；**二次确认**后删除并返回。
- **视觉规范**：新建与后续界面一律使用 **Variables（`$theme.*` 等）** 配色，保证深浅色切换一致；凡「是否」类布尔项均采用**设置同款开关**。

### 10. 我的日志

- **列表页**：日志列表；每条摘要上方展示**创建/最近修改时间，精确到秒**；右上角 **新建**为主题色加号胶囊。
- **详情/编辑页**：右上角 **完成**为主题色打勾胶囊；标题区下方展示**记录时间（精确到秒）**。
- **删除日志**：详情页**最底部**提供**删除日志**；**二次确认**后删除并返回列表。

### 11. 设置与外观

- **浅色为默认**，深色通过设置切换；支持**跟随系统**。
- **开关**：均为**左右拨动式**，与教务/其他布尔项区分，样式统一。
- **全局主题色**：可通过色值/取色器等形式配置，并驱动全局强调色（变量体系）。
- **界面元素变量化**：背景、表面、文字、边框、分隔、课表专用色等使用变量，支持**高级感深色模式**；课表内文字在深色下单独可读色（或阴影等策略）；存在 **`dark_mode`** 等与模式相关的变量逻辑（与原型说明一致）。
- **课表相关**：圆角、边框、阴影、隐藏周末、隐藏无课行、**学期开学日**、背景图与课表透明度等（与原型各区块对应）。
- **数据**：**本地备份**；**不包含云端备份**（已从需求与设置中移除，见 `prompts.md`）。
- **导出与分享**：支持导出/分享为图片或链接等（与设置或备份区文案一致）。
- **设置项排版**：外观相关第一组选项为**多行拆分**展示（非挤在同一行）。
- **课表管理**：设置页提供**课表管理**区块，列出当前本地多份课表。**默认「本人课表」（或等价主课表）行不显示删除**，不可在此删除；**其余课表行**名称右侧为主题色**删除（×）**，用于删除该份课表数据。**至少保留一份课表**时应拦截删除并提示；新建/重命名课表仍可在**导入与课表**相关流程中完成（与加号菜单、导入页一致）。

### 12. 个人中心状态

- 「我的」页已从当前主原型中移除，不作为主导航入口。
- 与个人中心相关能力在当前版本不作为主流程范围，后续如恢复需另立需求与页面。

### 13. 桌面小部件 / 小组件

- 多种尺寸小组件、下一节课信息、锁屏简要课表等（需求保留；实现可在后续版本或文案中说明通过系统小组件库添加）。

### 14. 删除入口覆盖自查（原型与实现）

以下实体在**当前主原型**中的删除路径如下；**未单列行内删除**的，默认通过详情或批量流程完成。

| 数据 | 删除入口（主流程） |
| --- | --- |
| 课程 | **课程详情**底部「删除课程」 |
| 考试/作业 | **考试与作业详情**底部删除 |
| 待办 | 列表**批量编辑**下删除；**待办详情**底部删除 |
| 日志 | **日志详情**底部删除（列表无单独删除按钮） |
| 日历手动标记 | 当日信息区下方**「擦除特殊日期标记」**（作用范围见上文「学期日历」中「当日标注展示与擦除」） |
| 全部课程 / 课表中的课 | 进入**课程详情**删除课程 |
| 课表多份（整表） | **设置 → 课表管理**中对**非默认主课表**行删除；**本人课表**行无删除入口；至少保留一份 |
| 课程详情内**已上传资料文件** | **编辑模式**下每行资料右侧主题色 **×** 删除 |
| 导入账号会话 / 本地备份文件 | 按设置与导入页既有设计，不强制详情页删除 |

## 三、暂缓实现与扩展稿

### 1. 课程评价社区（论坛）— 扩展原型

以下内容在独立文件 **`nju-timetable-forum-extension.pen`** 中保留高保真，**当前主应用不做实现**（见 `prompts.md` 第 18 条）：

- 课程社区列表、讨论详情、发布讨论（含匿名、点赞、收藏、评论、楼中楼、表情等迭代说明中的细节）。
- 主应用内已将**评价维度**与**资料**合并进**课程详情**，与社区列表解耦。

### 2. 已移除或不纳入当前范围

- **智能助手**（AI 问答、自动学习计划、考试周排表等）：已从功能与原型中移除（`prompts.md`）。
- **专注模式、习惯养成**及原「番茄钟/习惯」独立界面：已移除；若需专注计时可在后续版本单列需求。
- **课表整页背景图定制**：已从课表页方案中撤销，改为与全局一致的卡片/栅格风格。

### 3. 多端与云端（长期）

- Android / iOS / 平板 / Web **多端实时同步**、**云端自动备份与换机恢复**等：仍可作为长期目标，**当前文档以单机+本地备份为主**。
- **可选后端**：教务导入、OCR、公共节假日等若经服务端中转，设计与前端 DTO 对齐说明见 `docs/backend-design.md`（阶段 A 以客户端为数据真源；阶段 B 云同步另立评审）。

## 四、UI / 体验与实现约定

- **设计令牌**：颜色、分隔、开关轨道等优先使用 **Pencil Variables**（如 `$theme.bg`、`$theme.accent`、`$theme.toggleTrack` 等），深浅色切换时自动解析。
- **主原型画板（主文件）**包括但不限于：深浅色说明、导入与登录、周课表、课程详情（含底部删除课程、设置课程提醒、**资料编辑态行内主题色 ×**）、考试与作业、设置（含**课表管理**）、**日历**（含当日标注区与「擦除特殊日期标记」）、手动添加课程、功能页、全部课程、**待办**、**待办详情**（含底部删除）、我的日志、日志详情/编辑（含记录时间精确到秒、底部删除日志）、添加考试/作业、考试/作业详情（含底部删除）等。
- **无障碍与可读性**：深色模式下课表文字对比度需单独优化（已实现于变量方案，见 `prompts.md` 第 15–16 条）。

## 五、文档维护

- 产品迭代以 **`prompts.md`** 为变更日志、以 **`nju-timetable.pen`** 为界面真源；二者与本文档冲突时，应**先改原型与 prompts，再同步更新本节文档**。
- 工程约定：每次涉及产品/原型的改动，须按顺序更新 **`prompts.md`**（新增编号条目）、**`requirements document.md`**、**`frontend-functional-spec.md`**；详见项目 **`.cursor/rules`** 中的文档同步规则。
- **前端实现延伸（低重复）**：`docs/frontend-data-and-state-contract.md`（数据与状态契约）、`docs/frontend-engineering-layout.md`（工程结构与组件策略）、`docs/frontend-acceptance-matrix.md`（交互验收矩阵）；与 `docs/frontend-functional-spec.md` 末尾延伸章节互链。
