# 交互验收矩阵（测试视角）

本文档将 `requirements document.md` 与 `frontend-functional-spec.md` 中的关键行为压缩为**可勾选验收项**，便于自测与提测。不写实现细节；与「删除入口覆盖自查」表（需求第十四章）**互补**：此处侧重**流程与边界**，该表侧重**入口归位**。

**使用方法**：每条验收项下勾选「通过 / 失败 / 不适用」，失败需附截图与版本号。

---

## A. 导航与壳

| ID | 场景 | 期望 |
| --- | --- | --- |
| N1 | 冷启动 | 进入**课表** Tab，展示**本人/主课表**周视图 |
| N2 | 二级页 | **无**底部 Tab；顶栏返回位置下移，不遮挡状态栏 |
| N3 | 主 Tab | 仅 **课表 / 功能 / 待办** 三项可见 |

## B. 周课表

| ID | 场景 | 期望 |
| --- | --- | --- |
| T1 | 课表切换 | 标题区下拉可展开/收起；切换后网格刷新；**杀进程重进**恢复本人课表 |
| T2 | 加号菜单 | 贴近加号、顶层；含导入 / 设置或界面修改 / 全部课程；再点收起 |
| T3 | 课程设置 | 顶栏设置进入设置页 |
| T4 | 课程块 | 点击进入课程详情；块内主标题课名，**首行灰字教室** |
| T5 | 网格对齐 | 课程块与节次、星期列对齐；密集时不跨列错位 |
| T6 | 冲突 | 同一格多课**左右分栏**可读 |
| T7 | 周切换 | 切换教学周后数据与标题（月、第几周）一致 |
| T8 | `showOnGrid=false` | 周视图不显示；**全部课程**仍可见 |

## C. 课程录入与详情

| ID | 场景 | 期望 |
| --- | --- | --- |
| C1 | 手动添加 | 关闭「显示在课表」时校验规则与需求一致（如仅课名必填） |
| C2 | 保存 | 成功后返回上一页，课表/列表刷新 |
| C3 | 详情浏览 | 自添加字段已填；其余空态有引导文案 |
| C4 | 编辑态 | 铅笔进入，打勾提交并回到浏览态 |
| C5 | 资料 | 浏览可打开；编辑态行内 **×** 删除（建议确认）；与存储一致 |
| C6 | 提醒 | 自详情进入按**课程维度**配置；选项与需求一致 |
| C7 | 删除课程 | 底部删除 → 二次确认 → 返回并级联/提示符合产品规则 |
