# NJU Timetable 前端功能与整体框架文档

## 1. 文档目标与范围

- 目标：为后续前端代码实现提供统一的功能说明与架构边界，确保实现与 `requirements document.md` 和 `nju-timetable.pen` 一致。
- 当前阶段：仅定义功能和框架，不涉及任何代码实现细节。
- 范围：主应用（课表、功能、待办）及相关二级页面；课程社区扩展原型不纳入当前实现。

## 2. 产品信息架构

- 一级导航：`课表` / `功能` / `待办`
- 全局规则：
  - App 启动默认进入“我的课表（本人）”。
  - 二级页面均提供左上返回，且整体下移以预留系统状态栏区域。
  - 仅一级主页面保留底部胶囊导航，二级页面不显示。

## 3. 前端分层框架（功能视角）

- 展示层（View）：
  - 页面容器：每个原型画板对应一个路由页面。
  - 组件层：顶栏、胶囊按钮、列表项、开关、弹层、日历格、课程块等复用组件。
- 状态层（State）：
  - 全局状态：主题模式、主题色、当前课表、当前周、全局设置项。
  - 业务状态：课程数据、提醒配置、考试作业、待办、日志、日历标注。
  - UI 状态：下拉展开/收起、编辑态/浏览态、筛选排序态、弹窗态。
- 领域层（Domain）：
  - 课程域、课表视图域、待办域、考试作业域、日志域、设置域、日历域。
- 数据层（Data）：
  - 当前版本以本地数据持久化与本地备份为主。
  - 无云端数据库，服务器
  - 不包含云端自动同步、云备份恢复。

## 4. 路由与页面清单（按原型）

- `01 Import & Login`：导入与教务登录页
- `02 Week Timetable`：周课表主页
- `03 Course Detail`：课程详情页（含「设置课程提醒」入口；底部「删除课程」，需确认）
- `05 Exams & Homework`：考试与作业列表页
- `06 Settings & Appearance`：设置与外观页（含**课表管理**：列表中**本人课表**无删除，其余课表可 × 删除）
- `10 Semester Overview`：学期日历页
- `11 Manual Add Course`：手动添加课程页
- `12 Functions Hub`：功能聚合页
- `14 All Courses`：全部课程页
- `15 Todo`：待办列表页
- `15b Todo Detail`：待办详情页（含 `tdBody` 扩展区；底部「删除待办」）
- `16 My Logs`：日志列表页
- `17 Log Editor`：日志详情/编辑页（展示记录时间精确到秒；底部「删除日志」）
- `18 Add Exam/Homework`：新增考试/作业页
- `19 Exam/Homework Detail`：考试/作业详情页（底部删除本条）

注：不一定全部涵盖，后续可能会添加，注意添加的页面应该与现有界面风格/规约相同，可见requirements_documents.md

## 5. 关键交互流（用户点击后发生什么）

### 5.1 课表主流程（`02 Week Timetable`）

- 点击“我的课表”右侧下拉箭头：
  - 触发课表选择列表展开/收起。展开的列表读取已创建的课表有哪些并显示
  - 选择某个课表后刷新当前周网格数据与标题。
  - 退出重进后重置为“本人课表”。
- 点击顶栏“+”：
  - 弹出贴近按钮位置的下拉菜单（顶层覆盖）：
    - `导入课表`：跳转导入与登录流程页。
    - `界面修改`：跳转设置页。
    - `全部课程`：跳转全部课程页。
  - 再次点击“+”或空白区域，菜单收起。
- 点击顶部设置：跳转设置
- 点击课程块：
  - 进入课程详情页。
- 课表出现时间冲突时：
  - 同时段课程按左右分栏显示，不遮挡主标题信息。
- 点击周切换（周序或月份区域的操作控件）：
  - 切换教学周并刷新周网格课程。

### 5.2 导入与课程录入流程

- 在导入页点击“教务登录导入”：
  - 进入统一认证流程。
  - 登录成功后拉取并导入课表数据。
- 在导入页点击“OCR 导入”：
  - 进入截图解析流程。
  - 用户先选择截图所属教学周范围，再批量生成课程。
- 点击“手动添加课程”入口：
  - 进入手动添加页，支持填写课程名、教师、教室、节次、星期、周次规则。
- 在手动添加页点击“是否显示在课表”开关：
  - 开启：课程参与周课表展示。
  - 关闭：课程仅保留在全部课程/资料能力中，不在周视图展示。
  - 若关闭，只需填写课程名称（必填）任课老师（不必填，不用校验）
- 在手动添加页点击“保存”：
  - 校验必填项。（任课老师和上课教室不是必填项）
  - 保存成功后返回上一页并触发课表刷新。

### 5.3 全部课程与课程详情流程

- 在“全部课程”点击任一课程卡片：
  - 进入课程详情页。
- 课程详情页必填项通过读取添加课程时的数据自动填写，剩余项初始留空，由用户自行填写，并在留空时给出对用户填写哪些内容的引导
- 在课程详情点击右上“铅笔”：
  - 页面进入编辑态；图标切换为“打勾”。
- 编辑态点击右上“打勾”：
  - 提交当前编辑内容，回到浏览态，图标切换为铅笔。
- 课程详情中的资料区域点击“上传/打开资料”：
  - 上传本地文件并在列表中登记。
  - 浏览态：点击已上传项可直接打开。
  - **编辑态**：每条资料右侧显示**主题色小叉号**；点击移除该文件（建议二次确认），列表与本地存储同步更新。
- 点击“课程网站链接”：
  - 外部浏览器打开对应链接。
- 点击课程名称下方的「设置课程提醒」白底卡片：
  - 进入当前课程的提醒配置（时间粒度、提醒方式、节假日跳过等，与原先独立提醒页能力一致，数据按课程维度存储）。
- 点击页面底部「删除课程」：
  - 弹出二次确认；确认后删除该课程实体并返回上一页，刷新课表与全部课程等视图（关联考试/作业/待办按产品规则级联或提示）。

### 5.4 功能页分发流程（`12 Functions Hub`）

- 点击“日历”：
  - 进入学期日历页。
- 点击“考试与作业”：
  - 进入考试与作业页。
- 点击“日志”：
  - 进入日志列表页。
- 设置入口点击“设置/外观”：
  - 进入设置与外观页。

### 5.5 学期日历流程（`10 Semester Overview`）

- 自动读取考试与作业中的考试的日期并自行添加日期
- 自动读取每年的假期并自动添加
- 点击月份左右箭头：
  - 切换上月/下月，更新整月网格。
- 点击某一天日期格：
  - 下方当日信息区刷新（日期标题 + 当日待办 + 考试/放假/重要日等标注说明）。
- 当选中日存在可编辑类特殊标记时，在标注说明区域下方：
  - 展示「擦除特殊日期标记」按钮；点击后二次确认，清除用户为该日配置的手动标记（与自动同步的考试、只读假期数据的交互边界见 `requirements document.md` 学期日历条）。
- 点击“添加重要日（+胶囊）”：
  - 弹出新增重要日表单（日期、标题、备注），弹窗即可，非全页视图，风格与整体ui一致。
  - 保存后日期格按类型实色填充标注。
- 今日样式规则：
  - 今日为深灰描边突出，不使用整块实色填充。
- 如果一天同时是重要日/考试/放假中的多项，实色填充优先级重要日>考试>放假，未实色填充的改为同色描边，描边可多色描边环绕

### 5.6 课程提醒配置流程（自课程详情进入）

- 从课程详情点击「设置课程提醒」进入后：
  - 点击提醒时间胶囊（5/10/15/30 分钟等）：
    - 切换选中粒度，更新**该课程**的提醒策略。
  - 点击提醒方式项（通知/静音/震动/铃声）：
    - 可多选，点击选中，再次点击取消。
  - 点击“节假日跳过”开关：
    - 控制该课程在节假日是否仍触发提醒。
  - 对应课程：显示这个课程提醒是对应哪个课程

### 5.7 考试与作业流程（`05`/`18`/`19`）

- 在列表页点击右上“+新建”：
  - 进入新增考试/作业页。
- 在新增页点击“类型（考试/作业）”：
  - 动态切换表单字段：
    - 考试：时间、地点、关联课程、考试类型、备注。
    - 作业：DDL（支持多天时间规则）、关联内容、是否重复（滑动开关）。
- 提交新增后：
  - 返回列表并插入新项。
  - 若“关联考试与作业”开关已开启，同时同步到待办列表。
- 点击列表项：
  - 进入考试/作业详情页。
- 在详情页点击右上编辑：
  - 进入编辑态，保存后回详情浏览态。
- 点击底部「删除本条记录」：
  - 二次确认后删除该考试/作业，返回列表；若已同步待办则按联动规则更新。

### 5.8 待办流程（`15`/`15b`）

- 点击筛选 `全部/未完成/已完成`：
- 当日和全部不可以同时选中，但是可以和未完成/已完成其中之一同时选中
- 当日中含未标注日期的待办和所有作业
  - 实时切换列表数据集。
- 点击排序方式文字（默认/手动排序）：
  - 切换排序逻辑；选中项字体使用主题色。
- 点击“批量编辑”：
  - 列表行右侧操作切换为删除按钮（红底圆形垃圾桶）。
  - 点击删除可删除待办
  - 所有删除前弹出弹窗
- 在“手动排序”模式长按条目约 0.5s：
  - 进入拖拽排序态并更新序号。
- 点击条目右侧“完成”：
  - 条目变为已完成，标题划线并弱化；在“全部”中沉底显示。
- 点击条目本体：
  - 进入待办详情页。
- 在待办页底部点击“关联考试与作业”勾选圈：
  - 开启后自动纳入考试/作业项并持续同步新增项目。
- 在待办详情点击右上“编辑”：
  - 进入编辑模式。
- 点击待办详情底部「删除待办」：
  - 二次确认后删除并返回待办列表。
- 待办详情内点击“待办种类（单选）”：
  - 只允许三选一（持续时间 / DDL / 普通），下方时间字段按类型动态显示。

### 5.9 日志流程（`16`/`17`）

- 在日志列表点击右上“新建（+胶囊）”：
  - 进入日志详情页。
- 列表每条展示**时间精确到秒**（创建或最近更新时间，与数据字段一致）。
- 在日志详情页点击右上“编辑”：
  - 进入编辑模式
- 编辑模式下点击右上保存
  - 保存并返回日志列表，列表顶部显示新日志。
- 详情页标题区下方展示**记录时间（精确到秒）**。
- 二次编辑后增加展示上次修改时间（不显示在日志列表）
- 点击日志列表项：
  - 进入日志详情页进行查看或二次编辑。
- 点击底部「删除日志」：
  - 二次确认后删除该条并返回列表。

### 5.10 设置与外观流程（`06 Settings & Appearance`）

- **课表管理**：
  - 展示当前所有本地课表名称列表；**默认「我的课表」行不展示删除**，其余行右侧为主题色 **×**，删除该份课表。
  - 删除前二次确认；若删除后无剩余课表须拦截并提示至少保留一份。
  - 新建课表、编辑名称等仍走导入/课表主流程（非本区块必选能力，可与导入页能力合并实现）。
- 点击深色模式开关：
  - 切换 light/dark 主题变量解析结果。
- 点击“跟随系统”开关：
  - 主题模式由系统亮暗驱动。
- 点击“全局主题色”：
  - 打开色值/取色器，更新全局强调色变量。
- 点击“重新随机生成课表配色方案”：
  - 重新生成课程配色映射并刷新课表课程块色彩。
- 点击“隐藏周末/隐藏无课行”等开关：
  - 影响课表网格渲染策略。
- 点击“本地备份、导出与分享”：
  - 执行本地备份、导出图片或分享链接流程。

## 6. 前端常见约束（本项目落地版）

### 6.1 设计与主题约束

- 浅色模式为默认模式。
- 所有颜色/开关轨道/边框/文本色必须走变量体系，不允许页面内散落硬编码色值。
- 深色模式下课表文字可读性必须单独验收（对比度与阴影策略）。
- 课程色板要求低饱和、相邻可区分、不荧光刺眼，且冷暖分布均衡。

### 6.2 交互一致性约束

- 所有布尔型“是否”项统一使用滑动开关，不混用复选框样式。
- 右上操作按钮语义统一：
  - `铅笔` = 进入编辑态
  - `打勾` = 完成编辑并提交
- 胶囊按钮样式一致：主操作使用主题色填充，次操作使用描边或弱化样式。

### 6.3 数据与状态约束

- 任何“列表展示”必须与其数据源状态实时一致（新增、编辑、删除、完成态切换）。
- `课表课程`、`全部课程`、`课程详情`（含按课程的提醒配置）、`考试作业`、`待办`之间需维护关联一致性。
- 开启“关联考试与作业”后，新增考试/作业必须自动进入待办，不得需要二次手动同步。

### 6.4 页面与导航约束

- 一级导航仅保留：课表/功能/待办。
- “我的”页不在当前范围。
- 社区相关页面不在当前主应用范围（仅扩展稿保留）。

### 6.5 可用性与容错约束

- 关键表单（课程、考试、作业、待办）保存前必须做必填校验与格式校验。
- **删除类操作**（删除课程、删除待办、删除日志、删除考试/作业、擦除日历标记）须弹窗**二次确认**，且文案需说明不可恢复
- **仍无独立「删除」按钮的数据入口**（实现时核对）：日志列表行（依赖详情删除）、全部课程列表行（依赖课程详情）、课表块（依赖课程详情）。**资料文件**在课程详情编辑态用行内 **×** 删除；**多份课表**在设置「课表管理」删除（见需求文档自查表）。
- 外链跳转失败、导入失败、上传失败要有明确失败反馈与可重试路径。
- 长列表与高频交互（待办勾选、拖拽排序）需避免明显卡顿。

## 7. 非本期范围（明确不做）

- 课程社区（讨论列表、讨论详情、发布讨论）主应用不实现。
- 智能助手、专注模式、习惯养成不实现。
- 云端备份与多端实时同步不实现（长期目标保留）。

## 8. 实施顺序建议（仅功能框架）

- 第 1 阶段：路由骨架 + 全局状态（主题、导航、课表上下文）
- 第 2 阶段：课表主流程（周视图、加号菜单、课程详情跳转）
- 第 3 阶段：课程录入与导入（手动、教务、OCR）
- 第 4 阶段：功能分支（课程内提醒配置、日历、考试作业、日志）
- 第 5 阶段：待办体系与考试作业联动
- 第 6 阶段：设置与外观、备份导出、深色模式可读性收敛

## 其他约束条件
 
- 点按钮要有反应（加载、提示）
​
- 表单要判空
​
- 成功/失败要给提示
用户不知道成没成，会懵。

- 每个输入的框要给用户指引填写的内容
 
- 数据要先校验
​
- 不要啥都直接扔给后端
减轻后端压力，也减少请求。
 
- 加载状态要做
 
- 点击提交时，按钮禁用 + 显示加载
​
- 防止用户狂点多次，重复提交
这是最容易翻车的细节。
 
- 错误处理要考虑
 
- 网络失败
​
- 后端返回错误
​
- 数据为空
前端要显示提示，不能卡死、白屏。
 
- 注意页面跳转与权限
 
- 页面要可手动刷新，刷新页面状态还在
 
- 代码结构要清晰
 
- 页面分开
​
- 接口请求分开
​
- 工具函数分开
不要一个文件几千行，后期改不动。
 
## 可维护性（可以维护一个文档便于我理解）
 
- 变量名、函数名要看得懂
​
- 不要写一堆魔法数字、魔法字符串
​
