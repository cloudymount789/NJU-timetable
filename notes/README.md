# NJU Timetable — 架构与学习笔记索引

这些笔记面向**非专业背景**读者，用来对照仓库里的 `docs/` 文档，理解 AI 帮你整理的产品与工程边界，并为**你自己选技术栈**提供决策材料。

## 建议阅读顺序

1. **[文档地图与设计思路](./文档地图与设计思路.md)**  
   每份 `docs/` 文档在讲什么、它们之间谁听谁的、AI 整体是怎么「搭架子」的。

2. **[技术栈选型科普与对比](./技术栈选型科普与对比.md)**  
   客户端、本地存储、状态管理、可选后端等常见选项的原理、优缺点、业界替代方案。

3. **[架构质量标准与项目评价](./架构质量标准与项目评价.md)**  
   「好架构」一般看什么标准；对当前文档里这套设计的主观评价、风险与改进建议。

## 与官方工程文档的关系

- **产品/界面真源**：仍以根目录 `nju-timetable.pen`、`docs/requirements document.md`、`docs/prompts.md` 为准。  
- **`notes/`**：个人学习笔记与选型参考，**不替代** `docs/` 里的契约与规格；若与 `docs/` 冲突，以 `docs/` 为准。
